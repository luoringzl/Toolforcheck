# 核验工具

完全离线运行，用于批量审核职业技能等级认定申报人员的材料完整性及身份、学历、工作经历、企业信息一致性。

## 已实现

- 直接读取用户已经建立好的人员姓名一级子文件夹；不创建档案、不分组、不移动或重命名文件
- DOCX 正文/表格、电子 PDF 文本直接提取
- 扫描 PDF 和材料图片使用本机 RapidOCR；二寸证件照只检查竖版比例、彩色及半身正面构图，不进行文字识别
- 材料类型以正文、OCR文字或证件照视觉构图判定，文件名只用于显示，不作为分类依据
- 身份证图像质量门禁：模糊、强反光、严重旋转、尺寸不足时退回
- 身份证号码格式、出生日期、性别顺序码校验
- 加强身份证及申报表姓名/身份证号、毕业证校名/毕业时间/证书编号提取，并读取Word浮动文本框
- 逐行读取申报表工作经历表格：企业全称、职业、起止时间、证明人姓名和电话，并与工作证明、企业信息截图联动核验
- 兼容申报表 Word 纵向合并“工作经历”单元格，以及 PDF/OCR 将同一表格行拆成多行文字的情况
- 保留Word申报表真实行列结构并读取单元格内浮动文本框；表头、填写说明不再作为字段值
- 材料完整性审核固定列出基础及可配置材料类型，未触发的条件材料明确显示“不适用”
- 操作面板提供“批次辅助判断（可选）”：可预设高校在校/已毕业/已工作、最高学历和工作经历数量；保持默认时仍全部自动判断
- 可勾选本批次强制材料：工作证明、工作年限承诺书、学信网材料、企业信息截图、职业技能等级认定承诺书及其他材料；未勾选仍按业务规则自动判断
- 人工预设会写入Excel报告并与申报表实际内容再次核对，发现冲突时明确报告，不会静默覆盖材料内容
- 同类材料提交多份时，只要至少一份完整、清晰且可用，该材料类型即判定齐全；不合格副本标记为“材料不采用”
- 企业经营范围按每段工作对应的同一企业截图逐一核验，不跨企业借用经营范围
- 多份学历材料按最高层次、学信网优先、字段完整度和识别置信度择优；同层次冲突转人工复核
- 多份身份证出现不同有效号码时停止自动择优；关键OCR字段低置信度进入人工复核清单
- 输出“人工复核清单”和“字段识别依据”，保留字段值、置信度、文件及页码
- 企业简称硬性退回；企业全称与人员提交的企业信息截图逐项核对
- 输出人员总表、字段差异、工作经历、材料清单、退回清单、运行日志
- 所有处理均在本机完成
- 审核8类材料：《福建省职业技能等级认定申报表》、证件照、身份证、学历证明、工作证明、学信网学籍证明、工作年限承诺书、企业信息截图
- 多份身份证择优：只要存在一套清晰完整且有效的正反面即可
- 有工作经历时仅最后一段工作需要工作证明；2段及以上工作经历需要工作年限承诺书
- 每个工作经历企业均需企业信息截图，企业名称禁止简称
- 第一份工作必须从毕业次月或以后开始，且开始时年满16周岁
- 工作经历禁止重叠、允许断档
- 从事职业需符合企业经营范围，职能类岗位豁免
- 日期统一输出为 `2018.7`、工作区间统一为 `2018.7-2020.9`
- 兼容申报表出生年月只写年月，以及毕业时间填写完整年月日的情况；识别后统一按年月核验
- 区分学信网学籍、学历和学位验证材料，避免将学位证明误当作学历证明
- 学信网“在籍/在校”表示尚未毕业：院校取“学校名称”，申报表毕业时间按“预计毕业日期”核验；大专及以上在籍人员无需另交毕业证，已毕业人员仍须提交最高学历毕业证
- 识别职业技能等级证书的职业、方向、等级及证书编号
- 读取 Word/PDF 工作年限承诺书中的逐段经历，并核对声明年限与月份合计
- 工作年限承诺书姓名、身份证号与身份证核对，并将承诺书全部工作经历逐段与申报表核对，不得只填写最后一段
- Windows 安装向导始终显示安装目录选择页
- 提供“预审/正式审核”模式：预审允许工作证明暂未盖章，正式审核必须检测到单位公章
- 工作证明核对最后一段经历、证明人及电话、单位全称、在职“至今”和公章状态
- 工作证明规范陈述为“自×年×月至今，在××企业从事××职位×年”，姓名和身份证号与身份证核对，其余字段与申报表最后一段经历核对
- 身份证及其他材料图片允许横向或竖向提交；OCR自动尝试四个方向，退回依据仅为不完整、模糊、严重反光或严重歪斜等质量问题
- 操作界面采用分区卡片布局，统一清爽蓝灰配色、字体层级、运行状态和进度提示

## Windows源码运行

1. 安装 Python 3.11 或 3.12（安装时勾选 Add Python to PATH）。
2. RapidOCR、中文 ONNX 模型和OpenCV由依赖自动安装，无需另装OCR软件。
3. 在本目录执行：

```powershell
py -m pip install -r requirements.txt
py app.py
```

## 重要说明

这是规则完整、可运行的 MVP，并非联网背调或证件真伪鉴定工具。图片质量阈值和各类材料字段模板需要用脱敏的真实样本校准。软件不会对模糊身份证字符进行猜测或用其他材料自动补全。

## Windows安装包构建

正式安装包将Python运行环境、RapidOCR、ONNX中文模型及无界面OpenCV运行库全部打包，不要求使用者另装Python或OCR。冗余的Tesseract及语言模型已移除，并仅收集实际使用的ONNX后端。Windows构建机执行：

```powershell
.\build\windows\build.ps1
```

输出位于 `dist-installer`。仓库也包含可手动触发的Windows自动构建流程。
